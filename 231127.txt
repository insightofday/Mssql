SET NOCOUNT ON 명령을 사용하면 (dml작업시에도) SQL Server는 변경된 행 수를 반환하지 않습니다. 
이렇게 하면 네트워크 트래픽이 감소하고 응용 프로그램의 성능을 조금 더 향상시킬 수 있습니다. 
대부분의 경우에는 실제로 영향을 받은 행의 수를 반환할 필요가 없기 때문에 이 옵션을 활성화하여 사용하는 것이 좋을 수 있습니다.
--------------------------------------
UNION ALL은 SQL에서 두 개 이상의 SELECT 문의 결과를 결합하는 데 사용됩니다. 이를 사용하기 위한 주요 조건은 다음과 같습니다:

열의 수와 데이터 유형이 일치해야 함: 각 SELECT 문에서 반환되는 열의 수와 데이터 유형이 동일해야 합니다. 
즉, 동일한 위치에 있는 열의 데이터 유형이 호환 가능해야 합니다.

열 이름이나 별칭: 각 SELECT 문에서 열 이름이나 별칭은 첫 번째 SELECT 문을 따릅니다. 
열 이름은 같지 않아도 되지만 열의 순서와 데이터 형식이 일치해야 합니다.

중복된 행을 제거하지 않음: UNION ALL은 중복된 행을 제거하지 않고 모든 결과를 포함합니다. 
따라서 중복을 제거하지 않고 모든 결과를 합쳐야 할 때 사용됩니다.   UNION은 중복된 행을 제거하고 유일한 결과만 반환합니다.

각 SELECT 문은 독립적이어야 함: UNION ALL은 각 SELECT 문을 개별적인 쿼리로 처리합니다. 
따라서 각 SELECT 문은 독립적으로 실행될 수 있어야 하며, UNION ALL을 통해 결합될 수 있는 데이터를 반환해야 합니다.
-------------------------------------------
!!!union과 join은 목적 자체가 다르다!!!
UNION: select의 결과를 단일결과집합으로 결합,두 개 이상의 쿼리결과를 합침
JOIN:관련 키(조인조건)을 기반으로 테이블의 데이터를 결합하여 하나의 결과집합으로 반환,공유하는 칼럼이 있어야 함 
	테이블 관계를 기반으로 데이터를 추출(테이블을 연결)
